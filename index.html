Maximal Clique Enumeration Algorithms

Comparing Tomita, ELS, and Chiba-Nishizeki Algorithms

1. Introduction

Finding all maximal cliques in a graph is a fundamental problem in graph theory with applications in social networks, bioinformatics, and cybersecurity. This webpage presents a comparison of three efficient algorithms: Tomita, ELS, and Chiba-Nishizeki.

2. Algorithm Overview

Tomita Algorithm

Time Complexity: O(3ⁿ/³)

Description: Uses a depth-first search (DFS) approach similar to the Bron-Kerbosch algorithm. It introduces a pivoting technique to reduce unnecessary recursive calls and improve efficiency. This makes it optimal for dense graphs.

Best for: General graphs, especially dense networks.

ELS Algorithm

Time Complexity: O(d * n * 3ᵈ/³)

Description: A variation of the Bron-Kerbosch algorithm optimized for sparse graphs. Instead of treating all vertices equally, it orders them based on degeneracy, ensuring fewer recursive calls.

Best for: Sparse graphs, social networks.

Chiba-Nishizeki Algorithm

Time Complexity: O(a(G) * m)

Description: Focuses on edge-oriented traversal rather than vertex expansion. The algorithm is particularly useful for low-arboricity graphs, such as planar or biological networks.

Best for: Planar graphs, biological networks.

4. Best Algorithm Based on Graph Size

Graph Type Best Algorithm Reason
Small graphs (n < 10,000) Tomita Has optimal worst-case performance and is the fastest in practice.
Large, sparse graphs(e.g., social networks) ELS Its performance depends on degeneracy, making it much faster for sparse networks.
Planar graphs (e.g., road networks, biological networks) Chiba Optimal for low-arboricity graphs.
